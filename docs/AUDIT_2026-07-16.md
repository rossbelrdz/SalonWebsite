## 2. Por dominio

### 2.1 DBA / datos

**Bien:** FKs tenant, índices útiles, `eventKey` único en notificaciones, matrices en DB.

**Mal / riesgo:**
- Solo `db push` → sin historial de schema.
- Sin constraint de no-solapamiento de citas.
- Aislamiento multi-tenant solo en app (y a medias).
- Seed (antes) mutaba prod-like data; ya mitigado en 0.7.3.

**Mejoras:**
1. `prisma migrate` + `migrate deploy` en prod.
2. Exclusion constraint o serializable transaction al agendar.
3. Backups `pg_dump` + restore drill.
4. Resolver tenant desde sesión/membership, no solo env.

### 2.2 Seguridad / cyberops

**Bien:** bcrypt, cookie httpOnly, secretos cifrados en DB (si key fuerte), superadmin bypass consciente, credentials.md gitignored.

**Crítico:**
- Webhooks de pago sin firma.
- Superficie abierta de capture PayPal.
- Infra con puertos y secrets de demo.
- IDOR cancel + confirmación pública.
- Telegram link hijack.

**Mejoras:** firmas webhooks, rate limit + Turnstile real, revalidar sesión, headers, MFA obligatorio para superadmin.

### 2.3 DevOps / infra

**Bien:** healthchecks postgres/redis/app, tunnel profile, entrypoint Prisma (0.7.3), worker after healthy.

**Mal:** un solo compose dev usado como “live”; sin perfil prod; sin backups; Redis sin password; npm ci legacy race (mitigado parcialmente).

**Mejoras:**
- `docker-compose.prod.yml`: imagen `runner`, no bind-mount, no seed, migrate deploy, red interna.
- Secrets vía env/CI, fail-fast si defaults.
- Pin digests de imágenes.

### 2.4 App / producto técnico

**Bien:** F3–F7 funcionales, matrices admin, PWA/push/MFA/passkeys base, notificaciones BullMQ.

**Deuda:** booking races, PENDING sin TTL, TZ, multi-tenant real, validación Zod, authz fina en sidebar.

## 4. Cómo validar el fix Prisma (ahora)

```bash
docker compose up -d --build
docker compose logs app --tail 40   # debe verse: Prisma Client OK
# Abrir /admin/permisos y /admin/matriz-notificaciones
```

Si el volumen quedó corrupto:

```bash
FORCE_NPM_CI=1 docker compose up -d --build app worker
# o nuclear:
docker compose down
docker volume rm salon_web_node_modules
docker compose up -d --build
```
